Virtual Box 세팅

파일 > 환경설정 > 네트워크 > k8sNetwork 생성

hostIP         Port       virtualMachine : port(sshd)
host 127.0.0.1 101 ----> node1 10.100.101.22
host 127.0.0.1 102 ----> node2 10.100.102.22
host 127.0.0.1 104 ----> master 10.100.104.22

root로 로그인 후

# hosts 세팅
vi /etc/hosts 로 아래 입력 혹은 수정
10.100.0.104	master.example.com	master
10.100.0.101	node1.example.com	node1
10.100.0.102	node2.example.com	node2

# hostname 세팅
vi /etc/hostname 로 아래 입력 혹은 수정
master.example.com 혹은 node1.example.com으로 맞는 노드형태로 수정

Kubernetes 초기 설정

설치시 사용하는 명령은 root에서 실행한다

  • Installing kubeadm, kubelet, kubectl
    (kubeadm) 쿠버네티스 전체를 관리해주고 운영해주는 커맨드
    (kubelet) 쿠버네티스 컨테이너를 조작해주는 데몬
    (kubectl) 쿠버네티스에 명령어(ex. 파드조회, 웹서버 3개 실행 등등)를 수해할 때 사용하는 유틸
    • swap disable 처리 (마스터/노드)
    swapoff -a && sed -i '/swap/s/^/#/' /etc/fstab

    • 노드간 통신을 위한 브릿지 설정 (마스터/노드)
    cat <<EOF > /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF

#모듈 적재
modprobe overlay
modprobe br_netfilter

cat <<EOF > /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

#Apply sysctl params without reboot
sysctl --system

    • 방화벽제거 (마스터/노드)
    #방화벽 데몬 세팅 안되어있다면 무시
systemctl stop firewalld 
systemctl disable firewalld

    • 필수 패키지 apt-get (마스터/노드)
    apt-get update
apt-get install -y apt-transport-https ca-certificates curl

    • 구글 클라우드 퍼블릭 키 다운로드 (마스터/노드)
    curl -fsSLo /usr/share/keyrings/kubernetes-archive-keyring.gpg https://packages.cloud.google.com/apt/doc/apt-key.gpg

    • 레파지토리에 쿠버네티스관련 등록 (마스터/노드)
    echo "deb [signed-by=/usr/share/keyrings/kubernetes-archive-keyring.gpg] https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list

    • 레파지토리 업데이트 후 kubelet kubeadm kubectl 설치 (마스터/노드)
    apt-get update
apt-get install -y kubelet kubeadm kubectl
apt-mark hold kubelet kubeadm kubectl

    • 큐블렛 실행 및 인에이블 (마스터/노드)
    systemctl start kubelet
systemctl enable kubelet

    • systemd와 cgroup 맞춰 줘야함 (마스터/노드)
    #Kubernetes 1.22버전부터는 systemd와 cgroup 맞춰 줘야함
mkdir /etc/docker
cat <<EOF > /etc/docker/daemon.json
{
  "exec-opts": ["native.cgroupdriver=systemd"],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m"
  },
  "storage-driver": "overlay2"
}
EOF


systemctl enable docker 
systemctl daemon-reload
systemctl restart docker

  • master component 초기화 (😡반드시 master에서 실행‼️)
kubeadm init
# 혹은 kubeadm init --apiserver-advertise-address 10.100.0.104 --pod-network-cidr=10.100.0.0/24

  • kubectl 사용 권한 설정 (마스터/노드)

sudo chown $(id -u):$(id -g) $HOME/.kube/config
